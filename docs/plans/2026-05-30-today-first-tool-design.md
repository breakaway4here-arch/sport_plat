# sport_plat Today-first Tool Design

## Objective

把 `sport_plat` 调整成一个更容易第一次上手的移动端训练工具：首页默认进入 `训练`，优先回答“今天该做什么”，而不是先让用户进入计划配置页。

这次改造只重做展示层和默认入口，不改训练计划生成、动作替换、打卡保存、统计计算和 `localStorage` 数据结构。

## Product Position

- 不做营销页
- 不做 SaaS 产品首页
- 不引入服务端或数据库
- 继续保持“静态页面 + 浏览器本地数据”的形式
- 优先提升首次使用理解成本和每天使用效率

## Core Constraint

以下逻辑保持不变：

- `assignGoalsToDays()`
- `buildCombos()`
- `selectExercisesForDay()`
- `generateWeeklyPlan()`
- `replaceExercise()`
- `loadData() / saveData() / savePlan() / saveCheckin()`
- 自定义动作和统计口径

## Primary User Flow

目标用户既包括当前本人，也包括第一次打开页面的新用户。

首次打开时的目标路径：

1. 进入 `训练` 页
2. 立刻理解“还没有计划，需要先生成一周计划”
3. 点击主按钮跳到 `计划` 页
4. 选择训练目标、训练日、时长、器械并生成计划
5. 回到 `训练` 页直接看到今天训练内容
6. 完成动作后打卡

已有计划时的目标路径：

1. 打开即看到今天状态
2. 今天休息则看到恢复日提示
3. 今天有训练则直接看到动作清单和打卡入口
4. 需要改计划时再进入 `计划` 页

## Information Architecture

底部 tab 保持 4 个，但顺序调整为：

1. `训练`
2. `计划`
3. `打卡`
4. `统计`

默认进入逻辑：

- 优先恢复 `sessionStorage` 记录的上次 tab
- 如果没有记录，默认进入 `训练`
- 不再因为“没有计划”而默认进 `计划`

## Today-first Layout

`训练` 页承担首页职责。

### State A: No Plan

显示一个首开 Hero：

- `kicker`: `首次使用`
- `title`: `先生成你的第一周计划`
- `subtitle`: `选训练目标、训练日和器械后，今天该练什么会自动安排好。`

Hero 下方只保留一个主按钮：

- `去生成计划`

再给一个很短的 3 步说明：

- 选训练目标
- 选每周训练日
- 自动生成今天训练

不展示空的动作列表、进度条和统计卡片。

### State B: Plan Exists, Today Is Rest

Hero 保留恢复日状态：

- 今日状态
- 连续训练天数
- 本周训练天数
- 今日分钟数

下方只给一个轻量说明卡：

- 恢复日
- 建议散步 / 拉伸
- 可从这里跳到 `计划` 页查看本周安排

### State C: Plan Exists, Today Has Workout

页面顺序固定为：

1. 今日 Hero
2. 今日动作清单
3. 训练操作区

Hero 中展示：

- 今日状态
- 今日部位标签
- 完成度
- 预计时长
- 连续训练、本日计划动作数、额外项目数

动作区保留现有逻辑：

- 计划动作
- 额外项目
- 点击勾选
- 全部完成后打卡

训练操作区保留现有能力：

- 调整今日时长
- 添加额外项目
- 完成训练并打卡

## Plan Page Positioning

`计划` 页从首页降级为配置中心。

页面结构调整为：

1. 顶部显示当前周计划摘要
2. 中间显示训练条件配置
3. 下方显示当前计划结果

已有计划时：

- 默认优先让用户看到“当前计划摘要”
- 配置表单应弱化，不再像首屏首页
- “重新生成”和“修改条件”仍然可用

无计划时：

- 保持现有完整配置流
- 生成计划仍然是该页的主任务

## History Page

`打卡` 页继续保留：

- 月历
- 本月打卡列表

优化重点：

- 首屏先展示最近记录
- 空状态更轻
- 历史项对已删除自定义动作和未知动作 id 保持兼容

## Stats Page

`统计` 页继续保留现有统计逻辑，但把信息顺序改成：

1. 本周摘要
2. 本月摘要
3. 连续训练
4. 部位分布
5. 总览说明

目标是让新用户先读懂基础数字，再看趋势。

## Visual Direction

继续沿用当前深色移动端工具风格，不切回白底数据页。

改动重点：

- `训练` 页 Hero 更有层级
- 首开空状态更短、更清楚
- 主按钮层级更明确
- 同屏只保留一个绝对主按钮
- `计划` 页减少“默认先看到长表单”的压力

## Interaction Rules

- 首次无计划时不跳转、不弹独立 onboarding 页面
- 用 `训练` 页内嵌引导取代首开流程页
- 任何时候都不修改既有计划生成算法
- 任何时候都不破坏现有 `checkin` 数据
- Service Worker 仍然通过版本号更新，避免用户看到旧页面

## Acceptance Criteria

- 首次打开时，用户 10 秒内能明白下一步该做什么
- 首次打开时，主操作是“去生成计划”，而不是自己摸索 tab
- 已有计划时，用户打开就能看到今天该练什么
- 原有计划生成结果不变
- 原有打卡和统计数据不丢
- 默认入口从“计划优先”切换为“今日训练优先”
